# 受給者証・サービス更新期限管理システム 再現指示書

## 1. 目的

本システムは、障害福祉サービス利用者ごとの更新期限を管理し、期限切れや期限30日以内の対象者を見落とさないためのブラウザアプリである。

主目的は、計画相談・各サービス支給決定期間などの期限が近づいたときに、ダッシュボードと個人シート上で赤く目立つ表示にし、更新手続きの漏れを防ぐことである。

## 2. 基本方針

- 静的サイトとして作成する。
- GitHub Pages で公開できる構成にする。
- サーバーやデータベースは使わない。
- 利用者データはブラウザの `localStorage` に保存する。
- 個人情報をソースコードへ直接書き込まない。
- バックアップと移行は CSV エクスポート・インポートで行う。
- 画面は業務用として、見やすさ、期限の判別しやすさ、入力ミス防止を優先する。

## 3. 推奨ファイル構成

```text

tests/regression-check.js
README.md
システム再現指示書.md
```

## 4. 主要画面

### 4.1 ダッシュボード

画面を左右2分割にする。

左側:

- モニタリング月
- 当月モニタリング対象者を表示する
- 対象者がいない場合は「登録済みの利用者はいません」または「現在表示する対象はありません」と表示する

右側:

- 受給者証・サービス更新
- 期限30日以内または期限超過の対象者を赤く表示する
- 対象者ごとに、未完了の更新手続きタグを横並びで表示する

削除する表示:

- 「期限切れ・30日以内」
- 「60日以内に対応」
- 「期限管理」
- 「処理タスク」
- 件数カードだけの統計表示

### 4.2 個人シート一覧

- 登録済み利用者をカードまたは行で表示する。
- 「新規入力」ボタンから入力画面へ移動できる。
- 利用者を選択すると詳細画面を表示する。

### 4.3 個人シート詳細

個人ごとに以下を表示する。

- 更新時タスク
- 期限一覧
- 計画相談情報
- サービス期限情報
- 基本情報
- 備考
- 履歴確認

特に「期限一覧」は必須とする。

期限一覧では、以下が一目で分かるようにする。

| 表示項目 | 内容 |
|---|---|
| 何の期限か | 計画相談、訓練等給付費情報1、訓練等給付費情報2、介護給付費情報1、介護給付費情報2など |
| 期間 | 開始日から終了日まで |
| 残日数 | 期限まであと何日、または期限超過何日 |
| 判定 | 30日以内、60日以内、60日超、確認済、未入力 |

色分け:

- 期限超過、当日期限、30日以内: 赤
- 31日から60日以内: 注意色
- 60日超: 通常色
- 確認済: 緑
- 未入力: グレー

### 4.4 入力シート

入力項目は以下とする。

1. 個人情報
   - 氏名
   - フリガナ
   - 生年月日
   - 電話番号
   - 住所
   - 受給者証番号
   - 利用状態

2. 受給者証情報
   - 管轄する区
   - 自治体コード6桁
   - 障害者種別
   - 受給者証 有効期間 開始

3. 計画相談支援情報
   - 有効期間 開始
   - 有効期間 終了
   - モニタリング月
   - 利用者負担上限額

4. 訓練等給付費情報1
   - サービス種別
   - 決定有効期間 開始
   - 決定有効期間 終了
   - 使用事業所

5. 訓練等給付費情報2（共同生活援助）
   - サービス種別
   - 決定有効期間 開始
   - 決定有効期間 終了
   - 使用事業所
   - 区分種別

6. 介護給付費情報1
   - サービス種別
   - 決定有効期間 開始
   - 決定有効期間 終了
   - 使用事業所
   - 区分種別

7. 介護給付費情報2
   - サービス種別
   - 決定有効期間 開始
   - 決定有効期間 終了
   - 使用事業所
   - 区分種別

8. 更新時タスク
   - 書類作成
   - 役所送付
   - 本人受給者証交付確認
   - 受給者証の写し保存
   - 個人シートの更新

## 5. 入力ルール

### 5.1 日付

- 和暦入力を基本にする。
- 令和、平成、昭和、大正を選べるようにする。
- 年は2桁程度で入力できるようにし、6桁入力にならないようにする。
- 保存時は内部的に ISO 形式 `YYYY-MM-DD` へ変換する。
- 表示時は和暦で表示する。
- 保存後に再編集したとき、入力した年月日が `1月1日` に戻らないようにする。

### 5.2 モニタリング月

選択肢は以下とする。

- 1か月
- 2か月
- 3か月
- 4か月
- 5か月
- 半年

### 5.3 サービス追加欄

- 訓練等給付費情報1は1件だけ入力できればよい。
- 訓練等給付費情報2は1件だけ入力できればよい。
- 不要な追加ボタンは置かない。

## 6. 期限判定仕様

期限判定の対象は以下とする。

- 計画相談支援情報の有効期間終了
- 訓練等給付費情報1の決定有効期間終了
- 訓練等給付費情報2の決定有効期間終了
- 介護給付費情報1の決定有効期間終了
- 介護給付費情報2の決定有効期間終了

判定:

```text
期限日 - 今日の日付 = 残日数
```

- 残日数が30日以内: 赤く表示する
- 残日数が0日: 本日期限として赤く表示する
- 残日数がマイナス: 期限超過として赤く表示する
- 残日数が31日から60日: 注意表示
- 残日数が61日以上: 通常表示

「30日以内」には、期限超過と本日期限も含める。

## 7. 更新時タスク仕様

更新時タスクは1人1行にまとめて表示する。

複数タスクがある場合はタグを横並びにして省略表示する。

タスク:

1. 書類作成
2. 役所送付
3. 本人受給者証交付確認
4. 受給者証の写し保存
5. 個人シートの更新

5項目すべて完了した場合:

- 更新完了として扱う
- ダッシュボードの赤い対象から外す
- ただし、次回期限が30日以内になった場合は再表示できるようにする

初期状態:

- チェックは最初から入れない
- すべて未完了で開始する

## 8. 利用状態ステータス

利用者には以下のステータスを持たせる。

- 利用中
- 停止
- 終了
- 非表示

挙動:

- 利用中のみダッシュボードのアラート対象にする。
- 停止、終了は個人シートには残すが、アラート対象から外す。
- 非表示はダッシュボードと通常一覧から外す。

## 9. 履歴確認機能

個人シート詳細に履歴確認欄を設ける。

履歴に残す操作:

- 新規作成
- 個人シート更新
- 利用状態変更
- 更新時タスク完了
- 更新時タスク未完了へ戻す
- 期限確認済み
- 期限確認の取り消し
- CSVインポート

履歴には以下を保存する。

- 日時
- 操作名
- 詳細

## 10. バックアップ機能

バックアップ画面では以下を提供する。

- CSVエクスポート
- CSVインポート
- 登録件数表示

置かない機能:

- 全データ削除

理由:

業務アプリとして誤操作リスクが高く、通常運用では不要なため。

## 12. デザイン方針

- 業務画面として落ち着いた配色にする。
- 青を基調にしつつ、期限アラートの赤が最も目立つようにする。
- カードを乱用しない。
- ダッシュボードは左右2分割で、対象者を一覧できるようにする。
- 入力欄は重ならないように、横幅に応じて折り返す。
- 日付の「年」「月」「日」がつぶれないようにする。
- ボタンは大きめにし、誤操作を避ける。

## 13. 必須テスト

最低限、以下をテストする。

### 13.1 日付保存

- 計画相談支援情報の有効期間終了を令和8年6月20日にする
- 保存する
- 再編集する
- 令和8年6月20日のまま表示されること
- 1月1日に戻らないこと

### 13.2 30日以内アラート

- 今日から30日以内の日付を期限にする
- 保存する
- ダッシュボードで赤く表示されること
- 個人シート詳細の期限一覧でも赤く表示されること

### 13.3 期限超過

- 昨日以前の日付を期限にする
- 「期限超過 N日」と表示されること
- 赤く表示されること

### 13.4 更新時タスク

- 初期状態ではすべて未完了であること
- 5項目すべて完了すると、更新対象から外れること
- 次回期限が近づいたら再表示できること

### 13.5 ステータス

- 停止、終了はアラート対象から外れること
- 非表示は通常一覧にも出ないこと

### 13.6 CSVバックアップ

- CSVエクスポートできること
- CSVインポートで同じ内容を復元できること
- 個人情報をソースコードに直接保存していないこと

## 14. 公開時の注意

- GitHub Pages に公開する前に、ソースコード内に実在の個人情報が含まれていないか確認する。
- テスト用データを公開コードに直接入れない。
- 実データを移行する場合は CSV インポートで行う。
- ブラウザごとに `localStorage` が別であるため、別端末・別ブラウザではデータが自動共有されない。
- 共有運用が必要な場合は、将来的にサーバー保存または Google スプレッドシート連携を検討する。

## 15. 完成条件

以下を満たしたら再現完了とする。

- 利用者を新規作成できる
- 入力した日付が正しく保存・再表示される
- ダッシュボードが左右2分割で表示される
- 期限30日以内と期限超過が赤く表示される
- 個人シート詳細で「何が、いつまで、30日以内か」が一覧で分かる
- 更新時タスク5項目を管理できる
- 履歴確認ができる
- CSVバックアップと復元ができる
- GitHub Pages で表示できる
- 個人情報がコードに直接含まれていない
